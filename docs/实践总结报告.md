# 电商价格指数大数据分析系统 — 实践总结报告

> **课程设计大作业** | 权重：40%

---

## 摘要

本项目构建了一套完整的电商价格指数（CPI）大数据分析系统，涵盖数据生成、云存储、OLAP 计算、数据可视化四大核心模块，并通过 GitHub Actions 实现 CI/CD 自动化流水线。系统能够从原始交易数据中自动清洗脏数据，计算拉氏（Laspeyres）、派氏（Paasche）、费雪（Fisher）三种价格指数，并生成日度 CPI 趋势可视化图表。

---

## 一、实践内容

### 1.1 模拟数据生成（权重 15%）

- 生成三张核心表：分类维度表、商品信息表、价格明细表
- 目标数据量约 100MB，覆盖 365 天 × 5,000 SKU
- 故意注入约 2% 的脏数据（空值、负数、异常超大值），用于验证清洗流程

### 1.2 OSS 对象存储（权重 15%）

- 使用阿里云 OSS Python SDK（oss2）实现批量上传
- 支持断点续传和 MD5 完整性校验
- 密钥通过环境变量配置，确保安全性

### 1.3 ClickHouse 数据清洗与计算（权重 30%）

- 创建 ClickHouse 三表（MergeTree 引擎，按月分区）
- 实现 5 类脏数据过滤规则
- 支持 Laspeyres / Paasche / Fisher 三种指数计算方法
- 可回退为 pandas 本地模式（无需实际 ClickHouse 服务）

### 1.4 数据可视化（权重 20%）

- CPI 多指数对比折线图（拉氏/派氏/费雪三条曲线）
- 日均价格走势面积图
- 每日有效商品数量柱状图（数据质量监控）
- 支持 PNG / SVG 双格式输出

### 1.5 CI/CD 自动化流水线（权重 20%）

- 四阶段流水线：测试 → 数据生成 → 计算 → 可视化
- 使用 GitHub Actions 服务容器启动 ClickHouse
- 产物通过 Artifacts 归档，保留 7-30 天

## 二、技术难点与解决方案

### 2.1 大数据量下的内存控制

**难点**：100MB 价格明细表包含约 183 万行数据，一次性加载可能导致 OOM。
**方案**：采用逐日分块写入策略（Chunk-based Writing），每天生成当日的全部 SKU 价格后立即追加写入 CSV，全程内存占用 < 500MB。

### 2.2 脏数据识别与清洗

**难点**：原始数据包含多种脏数据类型，需准确识别并过滤。
**方案**：设计五级过滤规则链——空值 → 负数 → 超大值 → 过低值 → 缺货商品，每步记录过滤数量，输出清洗统计报告。

### 2.3 价格指数公式实现

**难点**：需正确实现经济学中三种不同的加权价格指数公式。
**方案**：手工推导小规模数值验证（如 2 商品 × 2 时期），确保公式实现与理论值一致，然后扩展到全量数据。

## 四、不足与展望

1. **未实现实时流处理**：当前为批量处理模式，可引入 Kafka + Flink 实现实时 CPI 计算
